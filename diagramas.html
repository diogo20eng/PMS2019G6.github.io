Introdução


Para este segundo Sprint, foi decidido utilizar uma abordagem Incremental em vez de uma abordagem Big Design Upfront, visto que pretendemos melhorar o projeto a cada Sprint, pois, a nosso ver é mais vantajoso.
Neste Sprint, foi-nos proposta a realização de vários diagramas, de forma a podermos representar de diversas maneiras o problemas sugerido. Utilizou-se, então, os diagramas de classes, diagramas de casos de utilização, diagramas de máquina de estados e, diagrama de sequência.

Diagrama de Máquina de Estados

Com este diagrama, pretendíamos, representar os vários estados no processo de criação de um projeto. Este pode ser Iniciado, Concluído, Arquivado e/ou Suspenso. Aquando, do estado, "Em Execução", pode passar ao estado "Concluído", se for terminado o desenvolvimento. Caso o Líder assim o decida, pode ser reaberto o projeto, passando, novamente ao estado "Em Execução".
Caso, o Líder decida arquivar, este passa para o estado "Arquivado".
Caso, existam atrasos no pagamento ou conflitos, este passará ao estado "Suspenso", passando pelos estados "Custos Suspenso" ou "Comunicação Suspensa", respetivamente.

Conclusão


Com a elaboração deste segundo Sprint, adquirímos uma maior prática no que toca à realização dos diagramas de casos de utilização, diagramas de classes, diagramas de máquina de estados e diagramas de sequência.
Para o próximo Sprint, tencionamos melhorar o que for necessário, como por exemplo, termos um maior número de casos de utilização, maior número de diagramas devido à falta de profundidade na análise do problema proposto.
Outro aspeto que pretendemos melhorar é a página de login que, de momento, não está completamente funcional e, pretendemos que assim o seja.
